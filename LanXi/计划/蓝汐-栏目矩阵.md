# 蓝汐栏目矩阵

> 用途：本文件是蓝汐小红书账号的栏目运营结构，用于把 30 天故事线沉淀为可长期复用的内容生产框架。后续生成文案、配图、发布日历、数据复盘时，应优先按本栏目矩阵归类。

## 栏目设计逻辑

蓝汐不是单纯的“头像素材号”，而是一个陪伴型原创 Q 版 IP。栏目需要同时满足四件事：

1. **建立人设记忆**：让用户记住蓝汐安静、温柔、轻微发呆、慢慢陪伴的气质。
2. **形成收藏价值**：头像、壁纸、短句、表情包都要能被保存、截图或转用。
3. **制造低门槛互动**：让用户可以留下名字、心情、ID、愿望、想看的动作或场景。
4. **沉淀可复盘数据**：每条内容都能归入固定栏目，方便后续判断哪些栏目更容易涨粉、收藏、评论或转化。

## 栏目总览

| 栏目 | 优先级 | 核心作用 | 发布频率 | 主要指标 |
|---|---|---|---:|---|
| 蓝汐日记 | 主栏目 | 建立角色日常与小世界连续性 | 每周 1-2 条 | 关注、评论、完读 |
| 一句软话 | 主栏目 | 提供情绪陪伴与收藏型短句 | 每周 1-2 条 | 收藏、转发、截图保存 |
| 名字小画 | 主栏目 | 建立粉丝共创与被记住感 | 每周 1 条 | 评论、关注、复访 |
| 头像领取 | 主栏目 | 承接头像需求，形成强收藏资产 | 每周 1 条或隔周 1 条 | 收藏、评论、主页访问 |
| 蓝汐小表情 | 辅助栏目 | 扩展日常互动和评论区传播 | 隔周 1 条 | 评论、转发、二创需求 |
| 场景故事 | 辅助栏目 | 提升画面叙事感与 IP 世界观 | 每周 1 条 | 收藏、关注、停留 |

## 栏目 1：蓝汐日记

| 项目 | 内容 |
|---|---|
| 栏目定位 | 蓝汐的日常记录、小世界推进和轻剧情入口。 |
| 用户需求 | 想持续了解这个角色；想看到她慢慢长大；想获得低压、轻陪伴感。 |
| 内容形式 | 单图、3-5 图小图组、轻剧情图组。 |
| 典型选题 | “蓝汐今天第一次出门”“蓝汐把一张纸收好了”“蓝汐在窗边坐了一会儿”“蓝汐今天也有点慢”。 |
| 画面规范 | 蓝汐为画面中心；背景可以是白纸、窗边、书桌、雨夜、云层、浅水倒影；低饱和、留白充足，不堆道具。 |
| 文案语气 | 短、轻、安静；像日记，不像广告。 |
| 互动方式 | “你今天发生了什么小事？”“你想让蓝汐明天去哪里？” |
| 禁忌 | 不写宏大剧情，不做强冲突，不让蓝汐变成高能量主角。 |

## 栏目 2：一句软话

| 项目 | 内容 |
|---|---|
| 栏目定位 | 蓝汐的情绪陪伴短句，适合做壁纸、收藏图和图文封面。 |
| 用户需求 | 想被安慰、想收藏一句不说教的话、想要低压情绪价值。 |
| 内容形式 | 单图壁纸、短句图组、头像/壁纸组合。 |
| 典型选题 | “慢一点也没关系”“我在呢”“今天先软软地过”“不用马上变好”“给你留了一个位置”。 |
| 画面规范 | 大面积留白；蓝汐可以小小站在角落或画面中央偏下；文字要少，不能喧宾夺主。 |
| 文案语气 | 温柔、克制、不鸡汤、不承诺解决现实问题。 |
| 互动方式 | “你想把哪句话存下来？”“今天想听蓝汐说什么？” |
| 禁忌 | 不制造焦虑，不使用过度煽情句式，不写“治愈一切”这类夸大表达。 |

## 栏目 3：名字小画

| 项目 | 内容 |
|---|---|
| 栏目定位 | 把粉丝的名字、ID、留言或小愿望轻轻放进蓝汐的小世界。 |
| 用户需求 | 想被角色记住；想拥有专属感；想参与共创。 |
| 内容形式 | 评论征集帖、to 签展示、名字小卡、ID 头像小样。 |
| 典型选题 | “把你的名字放在这里”“今天记住 3 个名字”“给几个名字留了位置”“你的 ID 想被画成什么感觉”。 |
| 画面规范 | 蓝汐旁边出现空白纸条、小卡片、书页或桌面；名字不能塞太满，优先保持画面干净。 |
| 文案语气 | 像邀请，不像抽奖；强调“慢慢画”“随机画”“先记住一些”。 |
| 互动方式 | “留下名字 + 一个小情绪”“留下 ID + 想要的颜色/场景”。 |
| 禁忌 | 不承诺每个人都会被画；不引导用户提供隐私信息；不出现真实敏感身份信息。 |

## 栏目 4：头像领取

| 项目 | 内容 |
|---|---|
| 栏目定位 | 提供可直接收藏、替换头像、Apple ID 头像或社交头像的蓝汐图。 |
| 用户需求 | 想要好看、统一、可爱但不张扬的头像；想用蓝汐表达自己的状态。 |
| 内容形式 | 头像合集、头像投票、节气/节日头像、动作头像。 |
| 典型选题 | “蓝汐头像 01：托脸发呆”“蓝汐头像 02：抱着小纸条”“蓝汐头像 03：看书”“蓝汐头像 04：雨天”。 |
| 画面规范 | 1:1 方图；圆形裁切安全；头部和眼睛清楚；纯白或低饱和浅蓝灰背景；不加复杂装饰。 |
| 文案语气 | 简短说明可领取/可保存，不做强营销。 |
| 互动方式 | “评论编号”“想要下次什么姿势？”“想要单人头像还是留白头像？” |
| 禁忌 | 不加入未经说明的配饰、动物耳朵、复杂边框或大面积文字。 |

## 栏目 5：蓝汐小表情

| 项目 | 内容 |
|---|---|
| 栏目定位 | 把蓝汐的日常反应做成表情包和评论区可用的小动作。 |
| 用户需求 | 想用蓝汐表达“我在”“慢一点”“抱抱纸”“发呆”“点头”等轻情绪。 |
| 内容形式 | 表情包九宫格、单个表情预览、评论区回复图。 |
| 典型选题 | “蓝汐说我在呢”“蓝汐点点头”“蓝汐抱着纸发呆”“蓝汐慢慢路过”“蓝汐收到”。 |
| 画面规范 | 动作要小，表情要克制；白底或极简背景；文字可作为后期版本，但首批建议先做无文字版。 |
| 文案语气 | 轻、笨拙、短，不做网络攻击感。 |
| 互动方式 | “你最需要哪个表情？”“评论一个想让蓝汐做的反应。” |
| 禁忌 | 不做阴阳怪气、攻击、嘲讽、暴躁、高能量表情。 |

## 栏目 6：场景故事

| 项目 | 内容 |
|---|---|
| 栏目定位 | 扩展蓝汐的小世界，让角色从纯白底进入低饱和日常场景。 |
| 用户需求 | 想看更有画面感、故事感、壁纸感的蓝汐；想知道她生活在哪里。 |
| 内容形式 | 3:4 图组、壁纸图、轻剧情分镜。 |
| 典型选题 | “蓝汐走进画纸之间”“蓝汐在雨夜窗边”“蓝汐捡到一张纸”“蓝汐坐在云边”“蓝汐在浅水倒影里看见自己”。 |
| 画面规范 | 允许前景/中景/远景层次；使用纸张、窗帘、雨滴、书页、台灯、云、浅水等轻道具；蓝汐身份和比例必须稳定。 |
| 文案语气 | 有叙事，但不沉重；像安静小故事，不像剧情大片。 |
| 互动方式 | “下一幕去哪里？”“纸上写什么？”“你想让蓝汐看到什么？” |
| 禁忌 | 不做复杂世界观堆砌，不加入其他角色，不用强冲突、战斗、恐惧、爆炸或高饱和场景。 |

## 首月发布配比建议

| 栏目 | 30 天建议条数 | 作用 |
|---|---:|---|
| 蓝汐日记 | 5-6 条 | 建立人设与连续追更 |
| 一句软话 | 5-6 条 | 拉收藏和情绪共鸣 |
| 名字小画 | 4 条 | 拉评论和粉丝关系 |
| 头像领取 | 4 条 | 拉收藏、关注和主页访问 |
| 蓝汐小表情 | 2-3 条 | 测试传播和评论区使用 |
| 场景故事 | 4-5 条 | 提升视觉质感和故事感 |
| 复盘/预告/投票 | 2-3 条 | 收集偏好并调整栏目 |

## 内容生产模板

每条笔记生产前，先填写下面 8 项：

```text
栏目：
本条目标：涨粉 / 收藏 / 评论 / 共创 / 视觉设定 / 复盘
用户需求：
画面形式：单图 / 图组 / 头像 / 壁纸 / 表情包 / 场景图
蓝汐动作：
文案核心：
互动钩子：
发布后记录指标：曝光 / 阅读 / 点赞 / 收藏 / 评论 / 关注 / 主页访问
```

## 首轮测试重点

第一轮不要平均用力，优先测试三个方向：

1. **头像领取**：判断蓝汐的视觉资产是否有收藏和使用价值。
2. **名字小画**：判断粉丝是否愿意留言、共创、等待下一批。
3. **一句软话**：判断蓝汐的陪伴型表达是否能带来收藏。

如果这三个方向表现稳定，再加大“蓝汐日记”和“场景故事”的连续更新比例。
