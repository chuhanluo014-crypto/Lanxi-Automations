# 候选选题池进入发布日历的标准流程

## 目标

候选选题池用于回答“哪些内容值得做”，发布日历用于回答“哪一天发哪一条”。两者分开维护，避免把未经筛选的想法直接排期发布。

## 输入文件

- `topic_candidates_v1.csv`：候选选题池，按《蓝汐-30天故事线表》的 Day 顺序组织。
- `LanXi/计划/蓝汐-30天故事线表.md`：30 天叙事主线，决定内容推进顺序。
- `LanXi/计划/蓝汐-栏目矩阵.md`：决定栏目配比和栏目目标。
- `case-analysis/合规检查表(已优化成skill名为publish-guard).md`：发布前合规检查依据。

## 输出文件

- `publish_calendar_v1.csv`：30 天发布日历。

## 标准流转

```text
候选选题池
  -> 按故事线顺序筛选
  -> 按栏目配比校验
  -> 选主选题和备选题
  -> 填入发布日历
  -> 生成内容草稿
  -> 四项发布前检查
  -> 发布
  -> 回填数据
```

## 从候选池填入发布日历的规则

### 1. 先看故事线

发布顺序以 `故事线顺序` 和 `故事线_day` 为准。不能因为某个候选题更像爆款，就提前打乱角色冷启动节奏。

判断方式：

- D1-D3：先建立角色是谁、长什么样、视觉记忆点。
- D4-D7：建立蓝汐的口吻、小世界和慢热性格。
- D8-D15：开始低门槛互动和小福利。
- D16-D23：固定栏目出现，让用户形成记忆。
- D24-D30：扩展小世界，交付第一轮福利，进入下一阶段。

### 2. 再看栏目配比

首月不追求平均发每个栏目，而是按冷启动优先级配置：

| 栏目 | 首月建议作用 |
|---|---|
| 蓝汐日记 | 建立人设和连续追更 |
| 一句软话 | 拉收藏和情绪共鸣 |
| 名字小画 | 拉评论、共创和复访 |
| 头像领取 | 拉收藏、关注和主页访问 |
| 蓝汐小表情 | 测试评论区传播和素材使用 |
| 场景故事 | 提升视觉质感和世界观记忆 |

### 3. 再选主选题

优先选择满足以下条件的候选题：

- `优先级 = P0`
- `状态 = 待检查` 或 `可进入内容生成`
- 与当前 Day 的 `故事线节点` 匹配
- 与当前 Day 的 `故事线栏目雏形` 匹配或可自然承接
- `合规风险` 可控

### 4. 需要备选题

发布日历中每个关键 Day 可以填 `备选topic_id`。备选题用于以下情况：

- 主选题生成效果不好
- 图片生产难度过高
- 内容和近期发布过于重复
- 发布前检查未通过

### 5. 四项检查通过后才能发布

发布日历中的 `发布前检查状态` 初始为 `待检查`。只有四项都通过后，才能改成 `可发布`。

四项检查：

| 检查项 | 通过标准 |
|---|---|
| 人设一致性 | 蓝汐仍然是安静、温柔、慢热、陪伴型原创 IP |
| 表达自然度 | 像自然小红书图文，不像 AI 文案、广告或说教 |
| 合规风险 | 不承诺治愈、不虚构经历、不诱导隐私、不用未授权素材 |
| 重复度 | 不和候选池、已发布内容、参考样本标题/画面过近 |

## 发布日历字段说明

| 字段 | 用法 |
|---|---|
| `calendar_id` | 发布日历行 ID |
| `day` | 对应 D1-D30 |
| `阶段` | 30 天故事线阶段 |
| `故事线节点` | 来自蓝汐 30 天故事线表 |
| `栏目` | 实际发布归属栏目 |
| `主topic_id` | 来自候选选题池的主选题 ID |
| `备选topic_id` | 可替换的候选题 |
| `发布主题` | 当天要做什么内容 |
| `内容形式` | 单图、图组、头像合集、表情包等 |
| `标题方向` | 标题生成的方向，不是最终标题 |
| `画面方向` | 配图或生图方向 |
| `互动钩子` | 评论区引导 |
| `来源类型` | 候选池 / 故事线原生内容 / 待补候选 |
| `发布前检查状态` | 待检查 / 需修改 / 可发布 |
| `生产状态` | 待生成 / 已生成草稿 / 已精修 / 已发布 |
| `发布后记录指标` | 发布后需要回填的数据 |

## 使用建议

1. 先打开 `publish_calendar_v1.csv`。
2. 按 `day` 从 D1 往后推进。
3. 对有 `主topic_id` 的行，回到 `topic_candidates_v1.csv` 查看完整候选信息。
4. 生成内容草稿后，填写四项检查结果。
5. 发布后回填曝光、阅读、点赞、收藏、评论、关注、主页访问和复盘备注。

## 重要边界

候选选题池不是发布承诺。进入发布日历后，也必须经过内容生成、人工精修和发布前检查。任何未通过检查的内容，都应改成 `需修改` 或 `暂不发布`。
